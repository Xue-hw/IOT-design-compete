# 团队分工与推进计划

> 设备统一口径：EYE 用于开发测试，Cube 用于最终演示且是 EYE 的功能子集，当前不要求同时在线。C3/AS7341 是共用光照子系统，不是第三个产品节点。EYE 屏幕未启用，Cube 无屏，由 P4 统一展示。

## 统一分工口径

| 角色 | 方向 | 核心目标 |
|---|---|---|
| A | S3 端 | 保持已烧录 S3 立方体稳定，负责采集、上传、提醒反馈 |
| B | P4 端 | 做出 P4 七寸屏展示节点，展示数据、大模型结果或系统状态 |
| C | 大模型 + 后端 | 接口、数据聚合、大模型复盘、提醒生成 |
| D | 前端 + 材料 | Web 看板、方案书、PPT、演示视频、提交打包 |

## A：S3 端

负责内容：

- 先完成 EYE 功能测试，测试后再统一下发 Cube 正式绑定和界面适配。
- 保留已稳定的 C3/AS7341 光照上传链路和物理来源标识。
- 确认当前主设备能输出光照、姿态、触摸/按键、专注计时、电量等字段。
- 固化 S3 上行 telemetry JSON 或提供真实样例数据。
- 不基于“所有 S3 都有屏幕”的假设增加功能。
- 输出 S3 演示步骤、字段说明和测试记录。

## B：P4 端

负责内容：

- 搭建 ESP32-P4 开发环境。
- 使用实际 7 寸屏做展示端。
- 优先展示三类内容：S3 最新状态、大模型复盘、提醒/建议。
- 与 C 约定 P4 拉取接口，优先走 HTTP。
- 输出 P4 工程说明、运行照片和演示步骤。

兜底目标：如果多媒体或复杂 UI 来不及，P4 至少作为“大屏展示终端”，稳定显示后端文本和状态。

## C：大模型与后端

负责内容：

- 实现 `POST /api/v1/telemetry` 接收 S3/P4 数据。
- 实现日报、曲线、提醒、状态接口。
- 调用云端大模型生成中文复盘和建议。
- 生成提醒，供 S3/P4/Web 拉取。
- 提供 mock 数据，保证 B/D 可以并行开发。

## D：前端与材料

负责内容：

- Web 看板展示设备状态、光照曲线、专注时间线、大模型复盘、提醒列表。
- 整理系统架构图、数据流图、硬件照片、运行截图。
- 编写方案书、PPT、视频脚本。
- 负责最终提交包目录和提交检查。

## 每日同步建议

- A 报告：S3 今天能输出什么数据，是否稳定。
- B 报告：P4 七寸屏今天显示了什么，是否能稳定联网/刷新。
- C 报告：接口是否可用，大模型是否能返回稳定结果。
- D 报告：看板和提交材料进度，还缺哪些截图/视频片段。
