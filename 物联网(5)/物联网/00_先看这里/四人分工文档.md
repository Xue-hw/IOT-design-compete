# FocusCube 四人分工文档

## 项目主线

当前项目按“四块闭环”推进：

```text
S3 立方体采集端
        ↓ telemetry
后端服务 + 云端大模型
        ↓ report / reminders / status
Web 看板 + P4 七寸屏展示端
        ↓
S3/P4 显示或响应提醒
```

核心原则：

- S3 已经烧录完成，尽量不大改，只做稳定、字段确认和演示整理。
- P4 是七寸屏展示端，要在最终视频里清楚出现。
- 大模型必须处理结构化感知数据，不能只是摆设。
- 前端和材料围绕“S3 采集 + P4 展示 + 大模型闭环”讲清楚。

## 成员 A：S3 端负责人

负责已完成的 S3 立方体采集端。

主要任务：

- 保持 S3 已烧录版本稳定。
- 确认可用采集字段：光照、姿态、触摸/按键、专注计时、电量。
- 提供 telemetry JSON 样例或真实数据导出方式。
- 保留本地提醒展示方式。
- 输出 S3 演示流程和测试记录。

验收标准：

- S3 能稳定展示或导出结构化数据。
- 演示过程中不频繁重启、卡死或断连。
- S3 数据能进入后端，或能被后端 replay 为真实样例。

## 成员 B：P4 端负责人

负责 ESP32-P4 七寸屏展示端。

主要任务：

- 搭建 P4 开发和烧录环境。
- 跑通 7 寸屏显示。
- 实现一个稳定展示页面，优先显示：
  - S3 最新状态。
  - 大模型复盘文本。
  - 提醒和建议。
- 与后端约定 P4 拉取接口。
- 输出 P4 演示 Demo、运行照片和工程说明。

验收标准：

- P4 工程可以独立编译和烧录。
- 7 寸屏在最终视频中可见、内容清楚、刷新稳定。
- P4 至少能显示一条后端数据或大模型结果。

## 成员 C：大模型与后端负责人

负责系统大脑：接收设备数据、聚合指标、调用云端大模型、生成报告和提醒。

核心接口：

```text
POST /api/v1/telemetry
GET  /api/v1/status
GET  /api/v1/report/daily?device_id=&date=
GET  /api/v1/reminders?device_id=&since=
GET  /api/v1/timeseries?device_id=&date=&metric=
GET/PUT /api/v1/config?device_id=
```

验收标准：

- 后端能接收 S3/P4 上传或模拟的数据。
- 大模型能基于结构化数据生成中文复盘。
- Web 和 P4 都能读取报告、提醒和状态。
- 演示时接口稳定，不依赖现场手工复制粘贴。

## 成员 D：前端、材料与总集成负责人

负责把作品展示出来，并保证提交材料完整、逻辑清楚。

主要任务：

- Web 看板：设备状态、曲线、复盘、提醒。
- 系统架构图和数据流图。
- 方案书、PPT、视频脚本和演示视频。
- 最终提交检查和打包。

验收标准：

- Web 页面能展示真实或准真实数据。
- 评委能看懂“大模型处理了什么、生成了什么”。
- 视频完整展示 S3、后端、大模型、Web、P4 七寸屏。

## 里程碑

### M1：最小闭环

S3 数据进入后端，后端调用大模型，Web 能展示复盘。

### M2：P4 屏幕接入

P4 七寸屏展示后端状态、大模型结果或提醒。

### M3：提交材料

方案书、PPT、视频、代码说明、测试记录进入 `06_提交材料`。
