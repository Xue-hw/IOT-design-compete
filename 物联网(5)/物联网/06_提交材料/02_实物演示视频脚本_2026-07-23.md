# FocusCube 实物演示视频脚本

> 版本：2026-07-23 v1
> 内部目标时长：3 分 30 秒至 4 分 10 秒。项目暂按 1–5 分钟制作，最终时长、分辨率、编码、大小和命名以报名系统/组委会模板为准。
> 原则：真实设备优先、一次讲清闭环、不伪装回放、不拍摄密钥或个人隐私。

## 1. 视频要证明什么

评委看完后应能明确回答：

1. 作品解决的是专注过程中的环境感知、交互和复盘问题；
2. EYE 与 C3 是同一基座上的两个协同节点，不是彼此重复的采集器；
3. C3 是原始光照唯一云端上传者，同时通过 TCP 把数据提供给 EYE；
4. EYE 结合姿态、会话和光环境做端侧派生分析；
5. 后端融合两个节点，生成规则提醒和云端大模型日报；
6. P4 与 Web 能展示结果；
7. 缺失数据有质量标记，不会用默认值污染统计。

## 2. 拍摄前置门槛

端到端主版本开拍前必须全部满足：

- [ ] EYE 与 C3 使用最终或候选最终固件；
- [ ] C3 真实 telemetry 获得 HTTP 200/201；
- [ ] EYE 合规 telemetry 获得 HTTP 200/201；
- [ ] C3→EYE TCP 真实数据可见；
- [ ] 后端可查询 EYE、C3 和 `focuscube-base-01`；
- [ ] P4 或 Web 已展示融合状态；
- [ ] 大模型日报已用多节点有效数据生成一次；
- [ ] 三档光环境至少预先彩排一遍；
- [ ] 所有手机、电脑和设备时间已校正；
- [ ] 画面中不出现 Wi-Fi 密码、令牌、学号、私人聊天或无关窗口。

若 schema 尚未完成，只能拍摄可复用的硬件特写、人物讲解和分模块功能素材；不得把分模块录像剪成“已完成端到端”的假象。

## 4. 旁白完整串词

以下为可直接录音的母稿，拍摄后应按实际画面删改，不能让旁白领先于功能证据。

> 这是 FocusCube，一套面向学习与办公场景的智能专注环境系统。它把端侧感知、局域网协同、云端融合和大模型复盘连接成完整闭环。
>
> 基座内包含两个独立节点。ESP32-C3 与 AS7341 持续采集光环境，是原始光照的唯一云端上传者；ESP32-S3 EYE 负责姿态、专注会话和端侧环境分析。两者共用五伏电源，但独立联网。
>
> EYE 使用板载 IMU 识别六面姿态，通过实体朝向降低操作负担。Focus 面用于专注，其他面承担状态和诊断入口；按键负责开始与结束会话。
>
> 现在开始一次专注会话。倒计时和会话编号由 EYE 本地状态机维护，即使网络短暂异常，基本交互仍可继续。
>
> C3 对 AS7341 采样进行质量检查，并把环境分为过暗、适宜和过亮。它一方面独立上传原始 lux 与标签，另一方面通过局域网 TCP 3333 把实时数据发送给 EYE。
>
> EYE 不重复上传原始光照，而是结合光环境、当前姿态和会话阶段，在端侧生成派生环境结论。这样既体现本地智能，也避免多个节点争夺同一数据的所有权。
>
> 两个节点分别上传结构化 telemetry。后端按同一安装实例和会话融合数据，同时保留 EYE 与 C3 的独立诊断状态，对用户则呈现统一的 focuscube-base-01。
>
> 对资源受限设备，我们允许非关键数据带质量标记降级上传。后端接受整包，但不会把缺失或占位值纳入统计和大模型输入，因此兼顾传输成功率和分析准确性。
>
> Web 页面用于历史和诊断，可以查看融合状态、数据来源、质量、光照时序和提醒。P4 七寸屏负责现场的一眼可读展示，中文字体、提醒滚动和真实后端请求均已完成适配。
>
> 后端把有效专注时长、光环境适宜比例和活动状态整理成结构化摘要，再由云端大模型生成简短、可执行的日报建议，缺失数据不会被模型擅自补全。
>
> FocusCube 不是简单的数据采集器，而是一个从端侧判断到云端复盘的可解释 AIoT 系统。谢谢观看。

## 6. 拍摄规范

### 6.1 实物

- 主画面使用横屏 16:9，建议 1080p 或以上；
- 固定相机和曝光，P4 屏幕避免灯带反光；
- AS7341 环境变化应缓慢、可复现，强光不要造成传感器饱和；
- 线缆整理后再拍，公共电源入口和两条支路可见；
- 重要按键动作至少拍两遍：全景一遍、手部特写一遍；
- 每个长镜头前拍 3 秒静止画面，便于剪辑。

### 6.2 屏幕录制

- 统一浏览器缩放和系统主题；
- 关闭通知、聊天窗口和个人书签栏；
- 公网 IP 可以显示，但 Wi-Fi 密码、令牌和服务端密钥必须打码；
- 日志字体确保 1080p 成片中可读；
- 后端、设备与页面的时间应尽量同屏或能通过时间戳对应；
- 不用开发者控制台手工改页面数值。

### 6.3 字幕与图形

- 设备名称固定为：`EYE`、`C3 + AS7341`、`P4`、`FocusCube 后端`；
- 原始光照箭头标注“C3 telemetry”，派生分析箭头标注“EYE telemetry”；
- `focuscube-base-01` 标注为“逻辑融合设备”，不能画成第四块硬件；
- 每个未完成能力若不得不出现，必须标注“规划项”；
- TinyML 不放在主功能字幕中，除非届时已有固件与评测证据。

## 7. 故障与备选镜头

| 风险 | 现场处理 | 可接受备选 |
|---|---|---|
| 公网瞬时不可用 | 保留本地交互拍摄，等待网络恢复后补录后端段 | 使用同一候选版本提前录制的真实成功结果，并标注“实测录屏”；不得伪装同一镜头实时发生 |
| P4 反光严重 | 关掉正上方灯、调整拍摄角度和曝光 | 使用屏幕录制/更近特写，但必须保留至少一个实物工作镜头 |
| lux 波动导致档位不稳定 | 使用遮光罩、稳定台灯和固定距离 | 展示两个稳定档位即可，第三档用独立真实镜头补充 |
| EYE/C3 TCP 临时断开 | 展示重连状态，重启本次会话 | 若重连本身稳定，可将其作为可靠性证据；否则不在主片中暴露未定位故障 |
| 大模型响应较慢 | 先展示请求，再切到其他功能，稍后回到结果 | 使用同一数据的真实已生成报告 |
| schema 尚未对齐 | 不拍“融合已完成”部分 | 只收集硬件和分模块素材，待对齐后补拍，不能直接提交该版本 |
