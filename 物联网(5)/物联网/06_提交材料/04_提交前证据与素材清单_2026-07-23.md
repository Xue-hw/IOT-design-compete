# FocusCube 提交前证据与素材清单

> 版本：2026-07-23 v1
> 用途：将“功能描述”转换为可复核的照片、日志、录屏和测试记录，供设计方案、演示视频和答辩共同使用。

## 1. 文件命名与目录建议

不建议直接把大体积原始视频提交到 Git 历史。可在团队共享空间建立：

```text
FocusCube_Submission_2026/
├─ 00_team_private/          # 身份和报名材料，不公开
├─ 01_product_photos/
├─ 02_eye/
├─ 03_c3_as7341/
├─ 04_backend_ai/
├─ 05_p4/
├─ 06_web/
├─ 07_end_to_end/
├─ 08_video_project/
├─ 09_design_document/
└─ 10_submission_receipts/
```

文件统一命名：

```text
YYYYMMDD_角色_模块_动作_结果_版本.扩展名
```

示例：

```text
20260724_A_C3_telemetry_HTTP201_fw-v2.jpg
20260724_C_backend_fused-status_focuscube-base-01.json
20260725_B_P4_daily-report_zh-cn_build-abc123.mp4
```

每份关键证据同时记录：设备、固件/commit、时间、网络、请求、预期、实际和结论。

## 6. 端到端“一镜到底”验收单

建议在正式拍摄前完成一次不剪辑彩排：

1. 开启公共 5 V 电源，EYE 与 C3 正常启动；
2. 确认两者联网且 C3→EYE TCP 已连接；
3. EYE 置于 Focus 面，按开始；
4. 后端看到同一 installation/session 的两节点 telemetry；
5. 改变光照，C3 lux/label 变化；
6. EYE 派生环境结论随有效输入变化；
7. 后端融合视图更新；
8. P4 与 Web 展示同一关键状态；
9. 制造一次非关键缺测，整包仍成功且页面显示降级；
