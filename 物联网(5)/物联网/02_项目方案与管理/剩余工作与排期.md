## 当前判断

不要再发散成旧的录音/拍照/文件传输项目。当前主线固定为：

```text
当前选中的 EYE 或 Cube 采集端
        ↓
后端 + 大模型
        ↓
Web 看板 + P4 七寸屏
        ↓
S3/P4 提醒展示
```

EYE 本地功能验收已完成，下一步是解决已烧录固件的后端地址差异，完成 Session/telemetry 端到端验收，再统一下发 Cube 的正式绑定和界面适配。当前 EYE/Cube 不要求同时在线。AS7341/C3 稳定上传链路继续保留，但 C3 不作为独立产品节点。P4 七寸屏是最终视频里必须出现的展示亮点。

## 兜底方案

- P4 复杂 UI 来不及时，改为大字体文本展示：设备状态、复盘、提醒三屏轮播。
- 大模型 API 不稳定时，后端返回最近一次成功结果或规则生成复盘。
- S3 上传不稳定时，使用 S3 真实导出数据 replay，视频中仍展示 S3 实物采集动作。
- Web 出现演示故障时，使用已部署看板的录屏和接口截图兜底。
