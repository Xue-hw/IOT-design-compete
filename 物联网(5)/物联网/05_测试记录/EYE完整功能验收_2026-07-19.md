# ESP32-S3-EYE 完整功能验收（2026-07-19）

## 1. 测试边界

- 角色：A 端实机验收。
- 设备：ESP32-S3-EYE v2.2，MAC 信息不作为公开设备身份。
- 串口：Windows `COM8`，115200 8N1，ESP32-S3 原生 USB Serial/JTAG。
- 固件：`esp32-s3eye-fullstack`，`9f0a290-dirty`，ESP-IDF `v5.5-dirty`，编译时间 2026-07-17 21:26:11。
- Flash：8 MB，DIO 80 MHz；PSRAM：8 MB Octal PSRAM 80 MHz。
- 维护入口：EYE SoftAP `http://192.168.4.1`。
- 原则：本轮不擦除、不重新烧录、不修改已验收固件。

本文不收录任何 Wi-Fi 密码、API Key 或未脱敏的原始启动日志。

## 2. 总体结论

EYE 的 HTTP 诊断链路、QMA6100P 六面识别、BOOT/HTTP 模式切换、SD、OV2640、I2S 麦克风、WAV 录音、CSI 记录、Dashboard、诊断页、30 分钟稳定性和多次复位恢复均通过。

未完成的核心项只剩多模态 Session 和后端上传闭环，原因是已烧录固件仍指向历史后端 `192.168.1.165:8000`，与 C 端当前局域网地址不一致。

## 4. IMU 与状态机

- IMU 实物为 QMA6100P，I2C 地址 `0x12`，`who_am_i=0x90`。
- `+X/-X/+Y/-Y/+Z/-Z` 六个方向逐面静置后均能正确识别。
- QMA6100P 没有陀螺仪，页面中 `gx/gy/gz=0` 或 N/A 为正常现象。
- 短按 BOOT 可在 `DETECT` 和 `ACTIVE` 间切换；再次短按可恢复。
- HTTP 模式切换同样可用。`led_pattern` 有约数秒异步更新延迟，稳定后 `ACTIVE=ON`、`DETECT=SLOW_BLINK`。
- 模式切换不会误启动音频、CSI 文件写入或 Session。

## 5. SD、摄像头和音频

### 5.1 SD

- SDMMC 1-bit 挂载成功，卡容量约 16 GB。
- 复位前后原有 CSI、WAV、JPEG 和 AVI 文件均未丢失。
- 新建 WAV 和 CSI CSV 后文件列表可立即读取。

### 5.2 摄像头

- OV2640 初始化、预热和单帧 JPEG 抓取正常。
- JPEG 无全黑、截断或明显花屏。
- 单次拍照后 `/status.free_heap_kb` 仅从约 8159 变为 8158，服务继续可用。

### 5.3 音频

麦克风指标实测：

| 环境 | samples | RMS | Peak | Avg Abs | error |
|---|---:|---:|---:|---:|---|
| 安静 | 22050 | 23.4 | 693 | 8.6 | `none` |
| 说话 | 22050 | 52.6 | 991 | 23.1 | `none` |

说话时 RMS 和平均绝对幅值明显上升，证明不是固定占位数据。完整 5 秒 WAV 录音和回放验收通过。

`/audio/metrics` 返回的 `duration_ms` 约为 300–330 ms，与页面所称的“1 秒”不完全一致，记为计时字段待核查项，不影响本轮麦克风功能判定。

## 6. CSI 与稳定性

- CSI 样本计数持续增长，RSSI 在实际无线链路中正常波动。
- `/start` 只启动 CSI 记录，不会误启动音频或 Session。
- 生成的 CSV 非空，包含多行数据，数值不是完全固定。
- 稳定性测试持续约 30 分钟，HTTP 始终可用，CSI 持续增长，无自动录音、无意外 Session、无持续堆下降。
- 稳定运行期间 `/status.free_heap_kb` 约为 8134；复位后约为 8158–8160。

## 7. 复位恢复

首次复位后曾出现 SoftAP 仍可见但 HTTP 暂时不可访问的未捕获瞬态现象。随后在串口监听下做受控复位，启动过程完整：

- SD、OV2640、QMA6100P、SoftAP、CSI、Video、Session、Upload 和 Web 任务均完成初始化。
- Web 在启动后约 1.8 秒注册 29 个 URI。
- STA 在短暂连接重试后约 6 秒获得 IP。
- 无 Guru Meditation、assert、watchdog 或 brownout。
- 又连续执行两轮复位，HTTP、SD、IMU 和空闲任务状态均在验收窗口内恢复。

结论：复位恢复通过，首次瞬态现象保留为后续压力测试观察项。

## 8. 已知不一致

1. `/status.board` 显示 `ESP32-S3-EYE-v1`，实物为 v2.2；当前只影响诊断文本。
2. 启动日志中出现短暂 STA 重试，但 `/status.sta_disconnect_count` 仍为 0；计数口径待核查。
3. 固件与 ESP-IDF 版本都带 `dirty`，完整源工程未进入共享仓库。
4. 启动日志会明文打印网络凭据，原始日志不得公开提交。
5. EYE 当前屏幕没有初始化证据，不得列为已实现功能。

## 9. 后端联调阻塞

实机 `/status` 显示：

```text
backend_ip=192.168.1.165
backend_port=8000
```

C 端 2026-07-19 回传的当时局域网地址为 `10.62.145.152:8000`，并计划开放 Windows TCP 8000 入站。两者地址不一致，所以本轮没有启动会自动上传的多模态 Session。

优先的无刷机方案是使用可配置路由器/OpenWrt，建立 `192.168.1.0/24` 专用网络，将 C 的后端电脑固定为 `192.168.1.165`。普通手机热点不支持所需的子网、静态路由或 DNAT，不能只靠 C 端修改后端代码解决。

如无法提供可控路由网络，则必须先取得与 `9f0a290-dirty` 实机一致的完整工程，备份 8 MB Flash，再修改后端配置并做全量回归。
