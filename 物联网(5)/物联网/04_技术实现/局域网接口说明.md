# 当前接口说明

本文件只描述当前比赛主线接口：S3/P4 与后端之间的结构化数据、报告和提醒接口。旧的音视频上传和文件传输路线不作为当前主线。

## 网络原则

- S3 尽量不大改，以已烧录版本可用的数据导出或上传方式为准。
- 后端提供稳定 HTTP API。
- P4 七寸屏优先通过 HTTP 拉取后端状态、报告和提醒。
- Web 看板使用同一组 API，避免为 P4 单独写一套数据格式。

## 后端 API

| 方法 | 路径 | 调用方 | 用途 |
|---|---|---|---|
| POST | `/api/v1/telemetry` | S3/P4/replay 脚本 | 上传结构化 telemetry |
| GET | `/api/v1/status` | Web/P4 | 获取系统最新状态 |
| GET | `/api/v1/report/daily?device_id=&date=` | Web/P4 | 获取大模型日报复盘 |
| GET | `/api/v1/reminders?device_id=&since=` | S3/P4/Web | 获取提醒列表 |
| GET | `/api/v1/timeseries?device_id=&date=&metric=` | Web | 获取曲线数据 |
| GET/PUT | `/api/v1/config?device_id=` | Web/调试 | 获取或修改配置 |

## telemetry 示例

### `valid` 兼容规则

- `imu.valid`、`focus.valid`、`power.valid` 为可选布尔字段，未发送时后端按 `true` 处理，兼容旧版本设备。
- 真实数据使用 `valid: true`，也可以省略 `valid`。
- 占位、模拟或暂不可用的数据必须显式发送 `valid: false`。
- 后端可以保存 `valid: false` 的原始数据用于调试，但不得将其计入日报、提醒、统计指标或传感器融合结果。
- 当前 AS7341 光照数据来自真实传感器，`light` 暂不增加 `valid`。
- P4/Web 遇到 `valid: false` 时应显示“等待真实数据”，不能把占位数值展示为真实测量结果。

```json
{
  "device_id": "focuscube-s3-01",
  "source": "s3",
  "ts": 1718000000,
  "light": {
    "lux": 235.6,
    "label": "suitable"
  },
  "imu": {
    "face": 2,
    "mode": "focus",
    "activity": 0.32
  },
  "focus": {
    "state": "running",
    "remaining_s": 940,
    "session_count": 3
  },
  "power": {
    "battery_pct": 78,
    "charging": false
  }
}
```

### AS7341 光照代理示例

当前没有可用 S3 实物时，AS7341 代理只提供真实光照；其余模块使用明确的无效占位结构：

```json
{
  "device_id": "focuscube-c3-proxy-01",
  "source": "c3-as7341-proxy",
  "ts": 1784039000,
  "light": {
    "lux": 360.72,
    "label": "suitable"
  },
  "imu": {
    "valid": false,
    "face": 0,
    "mode": "unknown",
    "activity": 0.0
  },
  "focus": {
    "valid": false,
    "state": "idle",
    "remaining_s": 0,
    "session_count": 0
  },
  "power": {
    "valid": false,
    "battery_pct": 0,
    "charging": false
  }
}
```

机器可读示例见 [`docs/api/examples/telemetry-as7341-proxy.json`](../../../docs/api/examples/telemetry-as7341-proxy.json)。

代理设备与未来真实 S3 必须使用不同身份：

```text
C3 + AS7341 代理: device_id=focuscube-c3-proxy-01, source=c3-as7341-proxy
真实 ESP32-S3:   device_id=focuscube-s3-01,       source=s3
```

后端、P4 和 Web 应通过配置选择当前展示设备，不能把 `focuscube-s3-01` 写死在多个源文件中。

### 光照阈值与标签

后端当前统一使用以下阈值：

| 条件 | 标签 |
|---|---|
| `lux < 200` | `too_dim` |
| `200 <= lux <= 500` | `suitable` |
| `lux > 500` | `too_bright` |

`/status` 直接显示设备上传的 `light.label`；`/reminders` 和日报适宜比例根据 `light.lux` 与上述后端阈值计算。设备端必须使用同一阈值，避免状态显示与提醒不一致。

## P4 七寸屏建议页面

P4 屏幕优先实现 3 个页面或 3 个轮播区：

- 设备状态：S3 在线状态、当前光照、专注状态。
- 大模型复盘：`report_text` 大字号展示。
- 提醒列表：最新 1-3 条提醒。

如果时间紧，先做单页大字体展示，不做复杂菜单。
