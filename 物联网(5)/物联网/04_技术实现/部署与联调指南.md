# 部署与联调指南

## 联调目标

跑通以下链路：

```text
S3 数据或 replay 样例
        ↓
后端 API
        ↓
大模型复盘和提醒
        ↓
Web 看板 + P4 七寸屏
```

## 1. 后端

后端至少需要提供：

- `POST /api/v1/telemetry`
- `GET /api/v1/status`
- `GET /api/v1/report/daily`
- `GET /api/v1/reminders`
- `GET /api/v1/timeseries`

建议先用 mock 数据启动，再接真实 S3 数据。

## 2. S3

S3 已烧录完成，联调原则是尽量不改固件。

优先顺序：

1. 如果 S3 已能 HTTP 上传 telemetry，直接接后端。
2. 如果上传不稳定，先导出真实 telemetry 样例。
3. 如果现场网络不稳定，用 replay 脚本向后端回放 S3 真实样例。

S3 演示时必须出现：

- S3 立方体实物。
- 至少一种传感数据变化。
- 数据进入后端或被后端 replay。

## 3. P4 七寸屏

P4 是当前展示重点。

优先实现：

1. P4 工程可编译、烧录、启动。
2. 七寸屏可显示固定文字。
3. P4 能请求后端 `GET /api/v1/status`。
4. P4 能显示 `report_text` 或提醒文本。

兜底 UI：

```text
FocusCube
S3: 在线
光照: 236 lux / 适宜
专注: 85 min

AI 复盘:
今天光照整体适宜，下午有一段偏暗，建议打开台灯。

提醒:
当前光线偏暗，建议打开台灯。
```

## 4. Web 看板

Web 看板与 P4 使用同一组后端接口。

最小页面包含：

- 设备状态。
- 今日指标。
- 光照曲线或专注时间线。
- 大模型复盘。
- 提醒列表。

## 5. 演示前检查

- [ ] 后端能启动。
- [ ] 大模型 API Key 可用，或已有兜底报告。
- [ ] S3 数据样例可用。
- [ ] Web 能显示 report。
- [ ] P4 七寸屏能显示后端数据。
- [ ] 视频脚本中的每一步都有负责人。
