# 固件计划与硬件配置

## 当前原则

- S3 是已烧录的立方体采集端，不做大规模改动。
- P4 是七寸屏展示端，是当前需要重点完成的端侧开发。
- 后端接口字段稳定后，S3、P4、Web 都按同一套接口接入。

## S3 端

当前目标：

- 保持稳定运行。
- 确认可输出字段。
- 提供 telemetry 样例或上传方式。
- 保留设备端提醒展示。

建议字段：

| 类别 | 字段 |
|---|---|
| 基本 | `device_id`, `source`, `ts` |
| 光照 | `light.lux`, `light.label` |
| 姿态 | `imu.face`, `imu.mode`, `imu.activity` |
| 专注 | `focus.state`, `focus.remaining_s`, `focus.session_count` |
| 电量 | `power.battery_pct`, `power.charging` |

## P4 端

当前目标：

- 跑通 P4 工程。
- 跑通 7 寸屏显示。
- 通过 HTTP 拉取后端数据。
- 展示 S3 状态、大模型复盘、提醒。

建议页面：

| 页面 | 内容 |
|---|---|
| 状态页 | S3 在线状态、当前光照、专注状态 |
| 复盘页 | 大模型 `report_text` |
| 提醒页 | 最新提醒列表 |

## 后端接口依赖

P4 端优先依赖：

```text
GET /api/v1/status
GET /api/v1/report/daily?device_id=focuscube-s3-01&date=YYYY-MM-DD
GET /api/v1/reminders?device_id=focuscube-s3-01&since=0
```

## 不做的事

- 不为了新功能重刷 S3 主流程。
- 不让 P4 同时承担采集、网关、多媒体和复杂本地 AI。
- 不再把旧的录音/拍照/文件传输作为当前比赛主线。
