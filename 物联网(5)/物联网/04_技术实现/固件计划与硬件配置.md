# 固件计划与硬件配置

## 当前原则

- 主设备分为测试用 ESP32-S3-EYE 与最终演示用 ESP32-S3 Cube；Cube 是 EYE 的功能子集，当前不要求同时在线。
- AS7341 所在的 ESP32-C3 是共用光照子系统，不是第三个产品节点；稳定上传链路继续保留。
- P4 是七寸屏展示端，是当前需要重点完成的端侧开发。
- 后端接口字段稳定后，S3、P4、Web 都按同一套接口接入。

## EYE / Cube 端

当前目标：

- 保持稳定运行。
- 确认可输出字段。
- 提供 telemetry 样例或上传方式。
- EYE 屏幕当前未启用，Cube 没有屏幕；提醒和状态先由 P4 展示。

建议字段：

| 类别 | 字段 |
|---|---|
| 基本 | `device_id`, `source`, `ts` |
| 光照 | `light.lux`, `light.label` |
| 姿态 | `imu.face`, `imu.mode`, `imu.activity` |
| 专注 | `focus.state`, `focus.remaining_s`, `focus.session_count` |
| 电量 | `power.battery_pct`, `power.charging` |

## P4 端

当前目标：

- 跑通 P4 工程。
- 跑通 7 寸屏显示。
- 通过 HTTP 拉取后端数据。
- 展示 S3 状态、大模型复盘、提醒。

建议页面：

| 页面 | 内容 |
|---|---|
| 状态页 | S3 在线状态、当前光照、专注状态 |
| 复盘页 | 大模型 `report_text` |
| 提醒页 | 最新提醒列表 |

## 后端接口依赖

P4 端优先依赖：

```text
GET /api/v1/status?device_id=<当前选择的设备ID>
GET /api/v1/report/daily?device_id=<当前选择的设备ID>&date=YYYY-MM-DD
GET /api/v1/reminders?device_id=<当前选择的设备ID>&since=0
```

## 不做的事

- EYE 功能测试完成前不提前绑定正式 EYE/Cube ID，也不假定二者同时在线。
- 不删除 `focuscube-c3-proxy-01` / `c3-as7341-proxy` 对应的稳定 C3 上传链路。
- 不让 P4 同时承担采集、网关、多媒体和复杂本地 AI。
- 不再把旧的录音/拍照/文件传输作为当前比赛主线。
